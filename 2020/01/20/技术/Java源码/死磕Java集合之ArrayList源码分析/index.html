    死磕Java集合之ArrayList源码分析

    简介

    ArrayList是一种以数组实现的List，具有动态扩展的能力。

    继承体系

    ArrayList实现了List，Cloneable，RandomAccess，Serializable等接口。

    • 实现了List，提供了基础的添加、删除、遍历等操作
    • 实现了Cloneable，可以被克隆
    • 实现了RandomAccess，可以被随机访问
    • 实现了Serializable，可以被序列化

    源码解析

    属性

    （1）DEFAULT_CAPACITY

    默认容量为10，也就是通过new ArrayList()创建时的默认容量

    （2）EMPTY_ELEMENTDATA

    空的数组，通过new ArrayList(0)创建时用的空数组

    （3）DEFAULTCAPACITY_EMPTY_ELEMENTDATA

    空的数组，通过new ArrayList()创建时用的是这个空数组。与EMPTY_ELEMENTDATA的区别是在添加第一个元素时会先初始化为容量为DEFAULT_CAPACITY（10）的空数组，然后再添加元素。

    Any empty ArrayList with elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA will be expanded to DEFAULT_CAPACITY when the first element is added.

    （4）elementData
    存放元素的地方，使用transient是为了不序列化这个字段。

    （5）size
    真正存储的元素个数，而不是elementData数组的长度。

    ArrayList(int initialCapacity)构造方法

    传入初始容量，如果大于0，就创建一个长度为对应大小的数组赋值给elementData。如果等于0，就使用EMPTY_ELEMENTDATA。如果小于0就报错。

    ArrayList()构造方法

    不传初始容量，使用DEFAULTCAPACITY_EMPTY_ELEMENTDATA空数组，会在添加第一个元素时扩容为默认的大小（10）。

    add(E e)方法

    添加元素到末尾，平均时间复杂度为 O(1)。

    （1）检查是否需要扩容；

    （2）如果elementData等于DEFAULTCAPACITY_EMPTY_ELEMENTDATA，则初始化的数组容量大小为DEFAULT_CAPACITY；

    （3）新容量是原来容量的1.5倍（oldCapacity + (oldCapacity >> 1)），如果新容量还是小于所需的容量，则以所需的容量为准；

    （4）创建新容量的数组，并把原数组拷贝到新数组；

    add(int index, E element)方法

    添加元素到指定位置，平均复杂度为 O(n)。

    （1）检查索引是否越界；

    （2）检查是否需要扩容；

    （3）把插入索引位置后的元素都往后挪一位；

    （4）在插入索引位置放置插入的元素；

    （5）大小加1；

    addAll(Collection c)方法

    将两个集合合并

    （1）拷贝c的元素到a数组中

    （2）是否扩容

    （3）把数组a的元素拷贝到elementData尾部

    get(int index)方法

    获取指定索引位置的元素，时间复杂度为O(1)。

    （1）检查索引是否越界，如果越界抛出IndexOutOfBoundsException异常

    （2）返回索引位置处的元素；

    remove(int index)方法

    删除指定索引位置的元素，时间复杂度为 O(n)。

    （1）检查索引是否越界；

    （2）获取指定索引位置的元素；

    （3）如果删除的不是最后一位，则其它元素往前移一位；

    （4）将最后一位置为null，方便GC回收；

    （5）返回删除的元素。

    可以看到，ArrayList删除元素的时候并没有缩容。

    remove(Object o)方法

    删除指定元素值的元素，时间复杂度为 O(n)。

    （1）找到第一个等于指定元素值的元素；

    （2）快速删除；fastRemove(int index)相对于remove(int index)少了检查索引越界的操作，可见jdk将性能优化到极致。

    retainAll(Collection c)方法

    仅保留此列表中包含在指定集合c中的元素，如果集合有所改变则返回true。

    （1）遍历elementData数组；

    （2）如果元素都在c中，则返回false；如果出现不在c中的元素，结束遍历，继续往下运行；

    （3）把之后还在c中出现过的元素都往前设置；

    （4）遍历完之后，w之前的元素都是两者共有的，w之后（包含）的元素不是两者共有的；

    （5）将w之后（包含）的元素置为null，方便GC回收；

    removeAll(Collection c)

    只保留当前集合中不在c中的元素。

    与retainAll(Collection c)方法类似，只是这里保留的是不在c中的元素。

    总结

    （1）ArrayList内部使用数组存储元素，当数组长度不够时进行扩容，每次加一半的空间，ArrayList不会进行缩容；

    （2）ArrayList支持随机访问，通过索引访问元素极快，时间复杂度为 O(1)；

    （3）ArrayList添加元素到尾部极快，平均时间复杂度为 O(1)；

    （4）ArrayList添加元素到中间比较慢，因为要搬移元素，平均时间复杂度为 O(n)；

    （5）ArrayList从尾部删除元素极快，时间复杂度为 O(1)；

    （6）ArrayList从中间删除元素比较慢，因为要搬移元素，平均时间复杂度为 O(n)；

    （7）ArrayList支持求并集，调用addAll(Collection c)方法即可；

    （8）ArrayList支持求交集，调用retainAll(Collection c)方法即可；

    （7）ArrayList支持求单向差集，调用removeAll(Collection c)方法即可；

    彩蛋

    elementData设置成了transient，那ArrayList是怎么把元素序列化的呢？

    查看writeObject()方法可知，先调用s.defaultWriteObject()方法，再把size写入到流中，再把元素一个一个的写入到流中。

    一般地，只要实现了Serializable接口即可自动序列化，writeObject()和readObject()是为了自己控制序列化的方式，这两个方法必须声明为private，在java.io.ObjectStreamClass#getPrivateMethod()方法中通过反射获取到writeObject()这个方法。

    在ArrayList的writeObject()方法中先调用了s.defaultWriteObject()方法，这个方法是写入非static非transient的属性，在ArrayList中也就是size属性。同样地，在readObject()方法中先调用了s.defaultReadObject()方法解析出了size属性。

    elementData为什么被transient修饰?

    elementData是一个缓存数组，它通常会预留一些容量，如果自己根据size序列化真实的元素，而不是根据数组的长度序列化元素，从而可以节省空间和时间。
